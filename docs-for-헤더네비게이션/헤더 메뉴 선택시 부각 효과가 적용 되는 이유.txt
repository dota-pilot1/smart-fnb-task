2번처럼 메뉴를 설정 하면 메뉴 클릭시에 활성화 메뉴가 내부적으로 설정 되고 
so 클릭한 메뉴에 활성화된 css가 적용됨

보충 설명:

[NavLink vs Link 차이]

- 일반 Link     : 클릭하면 URL만 이동. isActive 개념 없음. className은 문자열만 받음.
- NavLink       : 클릭하면 URL 이동 + 현재 URL과 to="" 값을 자동 비교해서 isActive를 계산해줌.
                  className에 함수를 전달할 수 있고, 그 함수의 인자로 { isActive }를 받음.

[동작 흐름]

  메뉴 클릭
    → URL 변경 (/products 또는 /devspec)
    → React Router가 현재 URL과 각 NavLink의 to="" 값을 비교
    → 일치하면 isActive: true, 불일치하면 isActive: false
    → navLinkClass 함수가 재실행되며 isActive 값에 따라 다른 CSS 문자열 반환
    → 해당 CSS가 NavLink에 적용됨

[navLinkClass 함수 역할]

  isActive가 true  → "... text-blue-600 after:bg-blue-600 ..."  (파란 텍스트 + 하단 밑줄)
  isActive가 false → "... text-gray-500 hover:text-gray-900"    (회색 텍스트)

[핵심 포인트]

  별도 상태 관리(useState, store 등)가 필요 없음.
  URL 자체가 "어떤 메뉴가 선택됐는가"의 상태이고,
  NavLink가 URL을 읽어서 자동으로 CSS를 결정하기 때문.

  현재 URL: /products   → 상품 메뉴 isActive:true  / 업무관리 isActive:false
  현재 URL: /devspec    → 상품 메뉴 isActive:false / 업무관리 isActive:true
