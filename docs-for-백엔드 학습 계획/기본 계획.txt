  🎯 프로젝트 전체 개요
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  프로젝트명 : DXShop (Digital Experience Shop)
  구성       : dxmall-frontend (Vite + React) + dxmall-server (Spring Boot 3)
  핵심 목표  : "현업에서 쓰는 기술 스택을 직접 구현해보며 깊이 있게 이해한다"

  이 계획서는 단순 To-Do가 아닌, 각 기능이 왜 필요한지·어떻게 배울지를
  담은 구현 가능한 위시리스트입니다. 우선순위별로 구분되어 있으며,
  Phase 1 → 2 → 3 순으로 진행하면 됩니다.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  📦 PHASE 1 — 기초 세팅 & 핵심 쇼핑몰 기능
  (목표: 돌아가는 쇼핑몰 뼈대 완성)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ─────────────────────────────────────────
  1-A. 백엔드 Spring Boot 기본 설정
  ─────────────────────────────────────────

  [ ] Spring Boot 3 + Gradle 프로젝트 구조 정리
      - DDD 레이어 분리: domain / application / interfaces / infrastructure
      - 패키지 네이밍 전략 수립 (com.dxshop.*)

  [ ] 데이터베이스 연동
      - Docker Compose로 PostgreSQL + Redis 로컬 환경 구성
      - Spring Data JPA 기본 CRUD 구현
      - Flyway 또는 Liquibase로 DB 마이그레이션 관리
      - QueryDSL 또는 jOOQ 도입 (복잡한 조회 쿼리 연습)

  [ ] 공통 기반 구현
      - 전역 예외 처리 (GlobalExceptionHandler, @RestControllerAdvice)
      - 공통 응답 포맷 (ApiResponse<T> wrapper)
      - Swagger / OpenAPI 3.0 문서 자동화
      - Logback + MDC로 요청 추적 로깅

  [ ] 인증 / 인가
      - JWT (Access Token + Refresh Token) 방식
      - Spring Security 필터 체인 이해
      - 소셜 로그인 (OAuth2 — 카카오 or 구글) 연동
      - 역할(Role): CUSTOMER, ADMIN, SELLER

  ─────────────────────────────────────────
  1-B. 상품 도메인 구현
  ─────────────────────────────────────────

  [ ] 카테고리 트리 구조
      - 계층형 카테고리 (재귀 트리 or adjacency list)
      - 카테고리별 상품 필터링 API

  [ ] 상품 CRUD
      - 상품명 / 가격 / 재고 / 이미지 / 설명
      - 상품 옵션 (사이즈, 컬러 등 — 옵션 조합 관리)
      - 이미지 업로드 (S3 or MinIO — presigned URL 방식)
      - 상품 검색 (Elasticsearch 또는 PostgreSQL Full-Text Search)

  [ ] 재고 관리
      - 동시성 제어 (Optimistic Lock or Redis를 활용한 재고 감소)
      - 품절 처리 및 알림 트리거

  ─────────────────────────────────────────
  1-C. 메뉴 시스템 (프론트)
  ─────────────────────────────────────────

  [ ] 헤더 네비게이션 메뉴
      - 카테고리 depth 기반 드롭다운 메가 메뉴
      - framer-motion으로 메뉴 애니메이션 (오픈/클로즈 스무스하게)
      - 모바일 햄버거 메뉴 (슬라이드 인/아웃)

  [ ] 사이드바 메뉴 (관리자용)
      - 아코디언 방식 중첩 메뉴
      - 현재 페이지 active 상태 표시
      - framer-motion LayoutAnimation으로 메뉴 expand/collapse

  📦 PHASE 2 — 실시간 기능 & 디자인 시스템
  (목표: 현업 수준의 실시간 기능 + 일관된 UI 완성)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ─────────────────────────────────────────
  2-A. 실시간 기능 — WebSocket
  ─────────────────────────────────────────

  [ ] 기술 선택: Spring WebSocket + STOMP + SockJS
      - 학습 포인트: WebSocket 핸드셰이크, STOMP 프레임 구조 이해

  [ ] 실시간 채팅 (1:1 문의)
      - 고객 ↔ 관리자 실시간 채팅
      - 채팅 이력 DB 저장 (ChatMessage 엔티티)
      - 읽음 / 안읽음 처리

  [ ] 실시간 알림 시스템
      - 주문 상태 변경 알림 (주문 접수 → 배송 중 → 배송 완료)
      - 재고 부족 알림 (관리자용)
      - SSE (Server-Sent Events) 방식도 비교 실습

  [ ] 실시간 재고 현황 (관리자 대시보드)
      - WebSocket으로 재고 변동 실시간 반영

  ─────────────────────────────────────────
  2-B. 실시간 기능 — Redis Streams
  ─────────────────────────────────────────

  [ ] Redis Streams 개념 학습
      - Kafka와의 차이점 이해 (경량 메시지 큐로서의 Redis)
      - XADD / XREAD / XREADGROUP / Consumer Group 실습

  [ ] 주문 이벤트 핸들링
      - 주문 생성 시 Redis Stream에 이벤트 발행
      - Consumer가 이벤트 소비 → 재고 차감, 알림 발송 처리
      - Dead Letter 처리 (실패한 메시지 재처리 전략)

  [ ] 이벤트 기반 아키텍처 실습
      - OrderCreatedEvent → InventoryService 소비
      - OrderCreatedEvent → NotificationService 소비
      - Spring @EventListener vs Redis Stream 차이 체감

  [ ] Redis 추가 활용
      - 세션 캐싱 (Spring Session + Redis)
      - 상품 조회수 / 인기 상품 랭킹 (Sorted Set)
      - 장바구니 캐싱 (Hash 자료구조)
      - 분산 락 (Redisson — 재고 감소 동시성 제어)

  ─────────────────────────────────────────
  2-C. 디자인 시스템 구축 (프론트)
  ─────────────────────────────────────────

  [ ] Design Token 정의
      - Color Palette (Primary, Secondary, Neutral, Semantic)
      - Typography Scale (font-size, line-height, font-weight)
      - Spacing Scale (4px 기반 그리드)
      - Border-radius, Shadow, Z-index 레벨 표준화
      - CSS Custom Property (Variables)로 토큰 관리

  [ ] 기초 컴포넌트 라이브러리
      - Button (primary, secondary, ghost, danger / sm, md, lg)
      - Input / Textarea / Select / Checkbox / Radio / Toggle
      - Badge / Tag / Chip
      - Avatar / Skeleton (로딩 상태)
      - Modal / Drawer / Sheet
      - Toast / Snackbar (알림 메시지)
      - Tooltip / Popover
      - Pagination / Breadcrumb

  [ ] 고급 컴포넌트
      - DataTable (정렬, 필터, 페이지네이션 통합)
      - ImageUploader (drag & drop, 미리보기, 멀티업로드)
      - RichTextEditor (Lexical 연동)
      - Chart (Recharts — 매출 그래프, 판매 현황)
      - DateRangePicker (기간 조회용)

  [ ] Storybook 연동 (선택)
      - 각 컴포넌트 문서화 및 인터랙티브 데모
      - Design Token 페이지 구성

  ─────────────────────────────────────────
  2-D. Framer Motion 실습
  ─────────────────────────────────────────

  [ ] 기본 애니메이션
      - motion.div, animate, initial, exit 이해
      - Transition 옵션 (tween, spring, inertia)

  [ ] 페이지 전환 애니메이션
      - AnimatePresence + React Router 조합
      - 페이지 fade-in / slide 전환 효과

  [ ] 리스트 / 카드 애니메이션
      - 상품 카드 hover 효과 (scale, shadow)
      - 상품 목록 stagger 애니메이션 (순차 등장)
      - useInView로 스크롤 트리거 애니메이션

  [ ] 제스처 인터랙션
      - drag를 활용한 장바구니 드래그 추가
      - whileHover / whileTap 마이크로 인터랙션
      - useSpring으로 물리 기반 애니메이션

  [ ] 레이아웃 애니메이션
      - layoutId로 상품 카드 → 상세 페이지 공유 요소 전환
      - 장바구니 수량 변경 시 숫자 애니메이션

  [ ] 고급 애니메이션
      - useMotionValue + useTransform (스크롤 패럴랙스)
      - SVG 패스 드로잉 애니메이션
      - Variants로 컴포넌트 상태별 애니메이션 관리

  📦 PHASE 3 — 결제 & 정산 & 관리자 페이지
  (목표: 실제 서비스 수준의 비즈니스 로직 완성)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  ─────────────────────────────────────────
  3-A. 결제 구현
  ─────────────────────────────────────────

  [ ] PG사 연동 — 토스페이먼츠 (권장) or 아임포트
      - 클라이언트 SDK 결제창 호출
      - 서버 사이드 결제 검증 (위변조 방지 — 금액 재검증 필수!)
      - 결제 성공 / 실패 / 취소 흐름 구현

  [ ] 주문 상태 머신 (Order State Machine)
      - 상태: PENDING → PAID → PREPARING → SHIPPED → DELIVERED → CANCELLED
      - 각 상태 전환 규칙 및 비즈니스 예외 처리
      - Spring StateMachine 라이브러리 또는 직접 구현

  [ ] 결제 취소 / 환불
      - 부분 취소 vs 전액 취소
      - 환불 정책 구현 (배송 전/후 환불 차이)
      - 취소 이력 관리

  [ ] 가상계좌 / 무통장 입금 (선택)
      - 입금 웹훅(Webhook) 처리
      - 입금 확인 후 주문 상태 자동 변경

  [ ] 결제 보안
      - Idempotency Key (중복 결제 방지)
      - HTTPS 강제화
      - 결제 모듈 격리 (보안 경계 명확화)

  ─────────────────────────────────────────
  3-B. 정산 구현
  ─────────────────────────────────────────

  [ ] 정산 개념 이해
      - 결제 금액 - PG 수수료 - 플랫폼 수수료 = 정산 금액
      - 정산 주기 설계 (일 정산 vs 주 정산)

  [ ] 정산 엔티티 설계
      - Settlement (정산 회차, 기간, 총 매출, 수수료, 정산액, 상태)
      - SettlementItem (주문별 세부 정산 내역)
      - 셀러별 정산 vs 플랫폼 전체 정산

  [ ] 정산 배치 처리 (Spring Batch)
      - @Scheduled 또는 Spring Batch Job으로 정산 집계
      - 기간별 주문 집계 → 정산 레코드 생성
      - 정산 완료 후 이메일 / 알림 발송

  [ ] 정산 보고서 / 엑셀 다운로드
      - Apache POI로 정산 데이터 엑셀 파일 생성
      - 관리자가 기간별 정산 보고서 다운로드

  ─────────────────────────────────────────
  3-C. 관리자 페이지 (Admin)
  ─────────────────────────────────────────

  [ ] 대시보드
      - 오늘 / 이번 주 / 이번 달 매출 통계
      - 신규 주문 / 신규 회원 현황 카드
      - 실시간 주문 피드 (WebSocket)
      - 매출 추이 차트 (Recharts — Line, Bar, Pie)

  [ ] 상품 관리
      - 상품 목록 (검색, 필터, 정렬)
      - 상품 등록 / 수정 / 삭제
      - 이미지 멀티업로드 (drag & drop)
      - 상품 옵션 동적 추가 UI
      - 재고 일괄 수정

  [ ] 주문 관리
      - 주문 목록 (기간별, 상태별 필터)
      - 주문 상세 (결제 내역, 배송 정보)
      - 주문 상태 변경 (배송 처리)
      - 환불 / 취소 처리 UI

  [ ] 회원 관리
      - 회원 목록 (검색, 필터)
      - 회원 상세 (구매 이력, 적립금)
      - 회원 등급 및 권한 관리

  [ ] 카테고리 관리
      - 트리 형태 카테고리 편집 UI
      - drag & drop으로 카테고리 순서 변경
      - 카테고리 활성/비활성 토글

  [ ] 쿠폰 / 프로모션 관리 (선택)
      - 쿠폰 생성 (정률 할인 / 정액 할인)
      - 쿠폰 사용 이력 추적

  [ ] 정산 관리
      - 정산 현황 조회 (기간별)
      - 정산 승인 / 반려 워크플로우
      - 정산 보고서 엑셀 다운로드

     → 각 Phase 구현 후 이 docs 폴더에 회고 파일 추가

     → JPA vs jOOQ
     → SSE vs WebSocket
     → JWT vs Session
     → 이런 비교가 면접과 실무에서 진짜 차별점이 된다.
