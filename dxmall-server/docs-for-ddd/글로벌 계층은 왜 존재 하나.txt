글로벌(global) 패키지는 왜 존재하나?
DDD의 일부가 아니라, 어느 도메인에도 속하지 않는 공통 코드를 모아둔 곳이다.

예를 들어:
- SecurityConfig, JwtProvider → 모든 API 인증에 사용
- GlobalExceptionHandler → 모든 도메인의 예외를 처리

이런 코드를 user/에 넣으면 나중에 product/, order/ 등
다른 도메인이 user/에 의존하게 되므로, 공통 영역으로 분리한 것이다.

이름은 global/, common/, shared/ 등 프로젝트마다 다르게 쓴다.
DDD 용어가 아니라 프로젝트 구조상의 관례다.
